## Recognition Rewards (No Cost)

These rewards cost nothing but create massive cultural impact.

### Weekly Quality Champion

**Frequency:** Every Monday standup  
**Cost:** $0  
**Selection:** Rotating senior picks one person from previous week

**Criteria (pick one):**
- Best code review feedback given
- Most improved code quality in PR
- Excellent refactoring example
- Helped teammate learn DDD/CA
- Caught important architecture issue

**Reward:**
- Announced in Monday standup with specific example
- Posted in #engineering Slack with ðŸ† emoji
- "Quality Champion" badge by name for the week
- Their work becomes teaching example

**Why it works:**
- Weekly frequency keeps momentum
- Specific examples teach everyone
- Costs nothing but means everything
- Everyone can win

### Golden PR Award

**Frequency:** As earned (typically 2-4 per month)  
**Cost:** $0  
**Selection:** Any senior can nominate

**Criteria (must meet 3 of 4):**
- Rich domain model with clear business behavior
- Excellent test coverage with meaningful assertions
- Uses ubiquitous language from domain
- Educational PR description explaining decisions

**Reward:**
- Special ðŸ† "Golden PR" label in GitHub
- Added to "Reference PRs" documentation
- Shared in Friday architecture discussion
- Creator presents approach in team meeting

**Why it works:**
- Creates learning library
- Makes good examples visible
- Recognized by peers
- Pride in craftsmanship

### Domain Expert Badge

**Frequency:** As earned (typically 1-2 per quarter)  
**Cost:** $0  
**Selection:** Senior consensus

**Criteria (must complete all):**
- Completed 3+ features in domain using DDD/CA correctly
- Contributed to Domain Dictionary
- Can teach domain to others
- Pass domain knowledge review

**Reward:**
- "Domain Expert" badge on GitHub profile
- Listed in domain README.md
- First choice for projects in that domain
- Invited to client domain discussions
- Can mentor others in that domain

**Why it works:**
- Clear mastery recognition
- Increased autonomy
- Career development
- Builds expertise depth

### Architecture Discussion Presenter

**Frequency:** Weekly (Friday 1-hour session)  
**Cost:** $0  
**Selection:** Volunteer or nominated by senior

**Format:**
- Developer presents interesting architecture decision
- Explains trade-offs and reasoning
- Team discusses and learns
- Decision documented

**Reward:**
- Lead the discussion (recognition)
- Their name on documented decision
- Builds reputation as thought leader
- Develops presentation skills

**Why it works:**
- Knowledge sharing is rewarding
- Develops communication skills
- Everyone learns together

### Monthly Shout-Out Wall

**Frequency:** Ongoing  
**Cost:** $0  
**Location:** #engineering Slack channel + office wall

**Anyone can post:**
- Thank teammate for help
- Recognize great code review
- Celebrate learning moment
- Share win

**Why it works:**
- Peer recognition
- Builds positive culture
- Costs nothing
- Everyone participates

## FAQs

### About Budget

**Q: What if we go over budget in a quarter?**  
A: We won't. Monthly individual awards are capped ($200+$100+$150 max). Team bonus is pre-allocated per quarter. Total can't exceed $15k.

**Q: What if we're under budget?**  
A: Unused funds roll to next quarter or fund team celebrations. Transparency maintained.

**Q: Can budget increase if we grow?**  
A: Yes. As team grows beyond 10 people, budget should scale proportionally.

### About Fairness

**Q: What if the same people always win?**  
A: 
- Quality Champion rotates (different senior picks each week)
- Team bonus ensures everyone benefits
- Multiple reward categories mean multiple winners
- If someone consistently wins, they're probably ready for promotion

**Q: What about new developers?**  
A: 
- Adjusted expectations during onboarding
- "Most improved" counts for Quality Champion
- Team bonus benefits everyone
- Growth recognized, not just perfection

**Q: How do we prevent favoritism?**  
A: 
- Transparent criteria
- Multiple nominators/reviewers
- Peer input valued
- Rotating selectors for Quality Champion

### About Gaming the System

**Q: What if someone games the metrics?**  
A: 
- Quality reviews by seniors, not just automated checks
- Focus on outcomes, not just outputs
- Team bonus ensures collective accountability
- 1-on-1 discussion if gaming suspected

**Q: What if code reviews become too lenient to be "mentors"?**  
A: 
- Mentor award requires quality feedback (documented)
- Kalpa spot-checks reviews
- Peer feedback on review quality
- Education > punishment if issues arise

### About Implementation

**Q: When do we start?**  
A: 
- Recognition rewards: Immediately (Month 1)
- Individual awards: Month 4 (after training starts)
- Team bonus: End of Q2 (Month 6)

**Q: Who decides winners?**  
A: 
- Quality Champion: Rotating senior (different each week)
- Golden PR: Any senior can nominate
- Mentor/Refactoring/Workshop: Senior consensus
- Team bonus: Objective metrics + transparent formula

**Q: Can we adjust the program?**  
A: Yes! Quarterly reviews allow adjustments based on what's working.
